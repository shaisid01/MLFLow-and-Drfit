Windows → EC2 → Docker Deployment Cheat Sheet
Step 0: Prerequisites

AWS EC2 instance (Ubuntu 22.04 LTS)

EC2 Key Pair: SpaceshipTitanic.pem

Security Group:

SSH (port 22)

HTTP (port 80)

Deployment folder on Windows:

C:/Shailaja/Prod_ready_AI_Project/Spaceship-titanic-ml/deployment_package/
├── app.py
├── Dockerfile
├── requirements.txt
├── model/          # model.pkl + preprocessor.pkl
└── src/            # preprocess.py
Step 1: Copy Deployment Package to EC2

Open Git Bash / MINGW64 on Windows:

scp -i "C:/path/to/SpaceshipTitanic.pem" -r "C:/Shailaja/Prod_ready_AI_Project/Spaceship-titanic-ml/deployment_package" ubuntu@EC2_PUBLIC_IP:/home/ubuntu/

✅ Replace EC2_PUBLIC_IP with your instance public IP.
✅ After this, your deployment folder will be on EC2 at /home/ubuntu/deployment_package.

Step 2: SSH into EC2
ssh -i "C:/path/to/SpaceshipTitanic.pem" ubuntu@EC2_PUBLIC_IP
Step 3: Install Docker on EC2 (Ubuntu)
sudo apt update
sudo apt install docker.io -y
sudo systemctl start docker
sudo systemctl enable docker
sudo usermod -aG docker ubuntu   # optional: run docker without sudo
docker version                    # verify installation
Step 4: Go to Deployment Folder
cd /home/ubuntu/deployment_package
ls

You should see:

app.py  Dockerfile  requirements.txt  model/  src/
Step 5: Build Docker Image
docker build -t spaceship-app .

-t spaceship-app → names your image

. → current folder

Ignore the BuildKit warning if it appears; it’s fine.

Step 6: Run Docker Container
docker run -d -p 80:5000 spaceship-app

-d → run in background

-p 80:5000 → map EC2 port 80 → container port 5000

Your API is now live at:

http://EC2_PUBLIC_IP/predict
Step 7: Test API

From your Windows machine:

curl -X POST http://EC2_PUBLIC_IP/predict \
-H "Content-Type: application/json" \
-d '{"HomePlanet":"Earth","CryoSleep":false,"Cabin":"B/0/P","Destination":"TRAPPIST-1e","Age":30,"VIP":false,"RoomService":0,"FoodCourt":0,"ShoppingMall":0,"Spa":0,"VRDeck":0}'

Expected output:

{"Transported": true}   # or false
Step 8: Keep Container Running After Reboot (Optional)
docker run -d --restart unless-stopped -p 80:5000 spaceship-app

Make Sure Flask App Binds Correctly

In your app.py:

if __name__ == "__main__":
    app.run(host="0.0.0.0", port=5000)

host="0.0.0.0" is essential so Docker exposes it outside the container.
